1. PROBLEM STATEMENT
================================================================================

Traditional recipe discovery is a time-consuming process where users struggle to:
  
  • Find recipes that match their available ingredients
  • Filter recipes by cooking time constraints
  • Discover recipes based on meal type preferences (Breakfast, Lunch, Dinner, Snack)
  • Get cuisine-specific recipe recommendations
  • Access reliable cooking instructions and video tutorials

CHALLENGES:
  - Users spend 10-15 minutes searching for suitable recipes online
  - Most recipe sites show irrelevant recipes that don't match ingredients
  - No intelligent way to combine available ingredients creatively
  - Limited personalization based on time and preferences
  - Manual YouTube searching for cooking video tutorials


================================================================================
2. PROPOSED SOLUTION
================================================================================

We developed an AI-Powered Recipe Recommender application that leverages:

  ✓ Google Gemini AI (Free API) - For intelligent recipe generation and ranking
  ✓ Streamlit Framework - For fast, interactive web interface
  ✓ Ingredient Normalization - To handle ingredient aliases and variations
  ✓ Rule-Based Filtering - To pre-filter recipes by cooking time, meal type, cuisine
  ✓ YouTube Integration - To generate direct links to cooking tutorials

KEY ADVANTAGES:
  - Generates recipes in real-time using AI
  - Uses available ingredients creatively
  - Respects cooking time constraints
  - Provides cuisine and meal-type based filtering
  - Shows AI-generated explanations for why each recipe is recommended
  - Includes YouTube video links for visual guidance
  - Free to use (Gemini API free tier)
  - No user registration or authentication required
  - Supports multiple languages for ingredient input

4. INPUT SPECIFICATIONS
================================================================================

The application accepts the following user inputs from the Streamlit sidebar:

INPUT 1 - AVAILABLE INGREDIENTS:
  • Type: Text area input
  • Format: Comma-separated list
  • Example: "chicken, onion, tomato, garlic, ginger, milk"
  • Required: Yes (minimum 1 ingredient)
  • Processing: Normalized and parsed by preprocessing.py
  • Supports: English and Hindi ingredient names

INPUT 2 - AVAILABLE COOKING TIME:
  • Type: Slider selection
  • Range: 5 to 120 minutes
  • Default: 30 minutes
  • Step: 5 minutes
  • Required: Yes (automatic default if not changed)
  • Usage: Filters recipes that can be prepared within this time

INPUT 3 - MEAL TYPE:
  • Type: Dropdown selection
  • Options: Breakfast, Lunch, Dinner, Snack
  • Default: Breakfast
  • Required: Yes
  • Usage: Narrows down recipe suggestions to meal type category

INPUT 4 - CUISINE PREFERENCE:
  • Type: Dropdown selection
  • Options: Indian, Italian, Chinese, Mexican, Thai, Japanese, Mediterranean, American
  • Default: Indian
  • Required: Yes
  • Usage: Filters recipes to specific cuisine style

The application generates and displays 3 AI-generated recipes with:

     • Display: Red button with play icon

  7. RECIPE INFO:
     • Cuisine: Selected cuisine type
     • Meal Type: Selected meal category

6. HOW TO RUN THE PROJECT
================================================================================

STEP 1: PREREQUISITES
  • Ensure Python 3.8 or higher is installed
  • Verify pip package manager is available
  • Get a free Google Gemini API key from:
    https://makersuite.google.com/app/apikey

STEP 2: CLONE THE REPOSITORY
  Open terminal/PowerShell and run:
  
  git clone https://github.com/VAMSHI-67/AI-Based_Recipe_Generator.git
  cd AI-Based_Recipe_Generator

STEP 3: CREATE VIRTUAL ENVIRONMENT (Optional but Recommended)
  
  # On Windows (PowerShell):
  python -m venv venv
  .\venv\Scripts\Activate.ps1
  
  # On macOS/Linux:
  python3 -m venv venv
  source venv/bin/activate

STEP 4: INSTALL DEPENDENCIES
  
  pip install -r requirements.txt
  
  This installs:
    - streamlit (web framework)
    - pandas (data processing)
    - google-generativeai (Gemini API)
    - python-dotenv (environment variables)

STEP 5: CONFIGURE API KEY
  
  Create a .env file in the project root:
  
  GEMINI_API_KEY=your_api_key_here
  
  Or copy from template:
  cp .env.example .env
  Then edit .env and add your actual API key

STEP 6: RUN THE APPLICATION
  
  # Using Python module execution:
  python -m streamlit run app.py
  
  # Or directly:
  streamlit run app.py
  
  Expected output:
  ```
  You can now view your Streamlit app in your browser.
  
  Local URL: http://localhost:8501
  Network URL: http://192.168.x.x:8501
  ```

STEP 7: ACCESS THE APPLICATION
  
  • Open your web browser
  • Navigate to: http://localhost:8501
  • You should see the AI Recipe Recommender interface
  • The Streamlit sidebar on the left has all input fields

STEP 8: TEST THE APPLICATION
  
  1. In the sidebar, enter ingredients: "bread, milk, sugar, dry fruits"
  2. Set cooking time: 15 minutes
  3. Select meal type: Snack
  4. Select cuisine: Indian
  5. Click "Find Recipes" button
  6. Wait for AI to generate recipes
  7. Review the 3 generated recipes in tabs
  8. Click YouTube links to watch tutorials


RUNNING ISSUES & SOLUTIONS:

Issue: "GEMINI_API_KEY not found in environment variables!"
Solution: Ensure .env file exists in project root with correct API key format

Issue: "ModuleNotFoundError: No module named 'streamlit'"
Solution: Run "pip install -r requirements.txt" to install all dependencies

Issue: "Permission denied" when running Streamlit
Solution: On Windows, run PowerShell as Administrator or use "python -m streamlit"

Issue: Port 8501 already in use
Solution: Use alternate port: streamlit run app.py --server.port 8502

7. DETAILED MODULE DESCRIPTIONS
================================================================================

MODULE 1: app.py (Streamlit Frontend)
  Purpose: Main user interface and application entry point
  
  Key Functions:
    • Page configuration and UI setup
    • Sidebar input collection (ingredients, time, meal type, cuisine)
    • Recipe display with tabs
    • Error handling and user feedback
    • YouTube link generation and display
  
  Execution Flow:
    1. Load environment variables from .env
    2. Import AI engine and utility modules
    3. Configure Streamlit page (title, icon, layout)
    4. Display sidebar with input controls
    5. On "Find Recipes" button click:
       - Validate ingredients input
       - Normalize ingredients using preprocessing
       - Call AI engine to generate 3 recipes
       - Display recipes in formatted tabs with all details
    6. Handle errors gracefully with error messages

MODULE 2: ai_engine.py (Google Gemini Integration)
  Purpose: AI-powered recipe generation using Gemini API
  
  Key Functions:
    • get_available_model() - Fetch available Gemini model
    • generate_recipe_with_gemini() - Generate single recipe
    • generate_multiple_recipes() - Generate 3 recipes
  
  AI Workflow:
    1. Check if GEMINI_API_KEY is configured
    2. Create prompt with user inputs and constraints
    3. Send prompt to Gemini API with model detection
    4. Parse JSON response from AI
    5. Validate recipe completeness
    6. Return formatted recipe dictionary
  
  Prompt Engineering:
    • Specifies cuisine and meal type
    • Lists available ingredients
    • Includes cooking time constraint
    • Requests JSON-formatted response
    • Ensures 60%+ ingredient usage
    • Asks for explanation of recipe choice

MODULE 3: recommender.py (Rule-Based Filtering)
  Purpose: Filter and rank recipes based on rules (if using CSV recipes)
  
  Key Functions:
    • filter_by_time() - Filter recipes by cooking time
    • filter_by_meal_type() - Filter by meal category
    • filter_by_cuisine() - Filter by cuisine type
    • calculate_ingredient_match() - Score ingredient overlap
    • rank_recipes() - Sort recipes by relevance
  
  Note: Currently, AI generates recipes dynamically, but these functions
        provide fallback filtering if recipe dataset is used

MODULE 4: youtube.py (YouTube Integration)
  Purpose: Generate YouTube search URLs for cooking tutorials
  
  Key Functions:
    • generate_youtube_search_url() - Create search URL
  
  URL Format:
    https://www.youtube.com/results?search_query={recipe_name}+{cuisine}+recipe
  
  Example Output:
    https://www.youtube.com/results?search_query=Palak+Chicken+Curry+Indian+recipe

MODULE 5: utils/preprocessing.py (Ingredient Normalization)
  Purpose: Parse and normalize ingredient input
  
  Key Functions:
    • normalize_ingredients() - Parse and standardize ingredient names
    • handle_aliases() - Convert common ingredient names
    • support_multiple_languages() - Handle English and Hindi ingredients
  
  Processing Steps:
    1. Split input by commas
    2. Remove extra whitespace
    3. Convert to lowercase for consistency
    4. Map aliases to standard names
    5. Return cleaned list of ingredients
  
  Example:
    Input: "Chicken, Onion, प्याज़, टमाटर"
    Output: ["chicken", "onion", "onion", "tomato"]

14. TROUBLESHOOTING GUIDE
================================================================================

PROBLEM: ModuleNotFoundError
  Cause: Missing dependencies
  Solution: pip install -r requirements.txt

PROBLEM: GEMINI_API_KEY not found
  Cause: .env file missing or incorrectly formatted
  Solution: Create .env with GEMINI_API_KEY=your_key

PROBLEM: "Streamlit seems to be running the script in an infinite loop"
  Cause: Streamlit reruns script on every interaction
  This is normal behavior, not an error

PROBLEM: Slow recipe generation
  Cause: Gemini API processing time or network latency
  Solution: Be patient, typically takes 3-10 seconds

PROBLEM: Recipes don't use all ingredients
  Cause: AI optimizes for quality; not all ingredients fit well
  Solution: This is intentional to maintain recipe authenticity

PROBLEM: YouTube links not working
  Cause: Network restriction or YouTube down
  Solution: Check internet connection or try again later
